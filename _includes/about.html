About

The idea involves a (group of) robot(s) that will gather at the central loading hub where the lockers will be loaded with shopping parcels that have to be delivered.

When a person desires that a delivery be performed, he/she uses a mobile application or website indicating he/she expects delivery at a certain location (within the apartment, university campus, IT park etc).

The delivery robot performs the delivery of the parcel to the person from the locker region.

A mobile application shows the progress of the delivery. The same application is used to authenticate the person to receive the parcel from the robot.

The delivery robot will have the features of an autonomous robot such as obstacle avoiding, future path planning, simultaneous localization and mapping (SLAM), computer vision for handling parcels of different shapes and sizes as well as for identifying the person the parcel is indended for.

The robot uses GPS and SLAM for localization. It uses BlueTooth, WiFi or NFC for communication with external machines such as elevators. It uses radar for sensing distances to the obstacles around it. A camera with a stepper motor arrangement is used to take pictures fed into computer vision programs to gauge the distances to obstacles, form factor of obstacles, edges of parcels, identifying the intended receiver of the parcel etc. The robot has an on-board Single Board Computer device for decison making. A microcontroller such as the Arduino Mega, is used to interface the sensors to the Single Board Computer.

The robot uses learning algorithms to improve path planning.

The otherwise completely autonomous robot, can be monitored as it will always be connected to the Internet through WiFi, if available or through GSM using which it optionally uploads real-time logs to a central server.
